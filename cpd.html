Automated Printing

A Python Script

About

This is a Python Script I wrote to increase my producivity and workflow. We have one customer with high order volumes and consistent job requests. Due to the natue of the orders, automating most of the process was a no brainer.


Main Project Points

Download Phase

• Gathers Emails

• Reads the HTML form within the Email

• Identifies the Google Drive Links

• Downloads the Google Drive Files

• Groups Everything by Order Number

• Converts Files to Postscript

• Creates JSON file with Job Specifications

• Merges Postscript files if needed


Printing Phase

• Supports Bulk Printing via Order Numbers

• Picks the Settings for the Printer for you

• Manual Inseration of Set Count if unable to determine

• Custom Banner Sheet's with Job Specs's, Files, and Page Counts

• Job Error Checking

• Injection of Printer Job Language (PJL) commands into postscript files

Timeline / Proccess

  • Python

    After doing some reaserch at differnet languages and their aviable libraries, and which languages I was most comfortable in, I settled on using Python as the language for my program.

    Im also using these libraries:

    • PdfFileReader from PyPDF2 for page counts

    • glob for getting the list of files and folders

  • Gmail API

    All of our jobs come to us via Email to a dedicated Gmail account. The Python Script, using the Gmail API looks through the Unread Emails (under normal use case thier is no human interaction with the email account) and downloads the Subject and Contents into a text file. I found a premade script (Robulouski's "gmail_imap_dump_eml.py") that shows the Gmail API in action, using that and modified it to my needs.

  • Google Drive API

    The Emails are formatted in an html for that is consistent between them all. After the contents of the Email are recivied, the Script parses through the information, pulling out the Google Drive Links, thier Google Drive ID's and cleaning it up for the Google Drive API. This StackOverFlow Response helped me understand the Drive API and adapt it to my needs.

  • Job Organization

    All of the Jobs are sorted into folders in the format Order Number + Subject Line. The Order Number is referenced from within the body of the email, and the Subject Line takes the first 45 valid characters for windows.

    All the folders contain the Email in a text file, A JSON parameter file, All the PDF's for the job and a PostScript Folder with all the pdfs converted to postscript.

    In addition, as needed their is a merged file of all the files if the job needs it, and during the printing process, and Banner Sheet file is created, and another folder is created with print ready postscript files,

    In the root, thier is also a folder with the default PJL (Printer Job Language) parameters for our Xerox D110. These get added to all the postscript files and it is modified to fit the parameters of the requested job.
